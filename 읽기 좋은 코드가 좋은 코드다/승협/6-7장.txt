# 6장 정리

## 명확하고 간결한 주석 달기

1. 모호한 대명사를 피한다. 이것, 저것을 이런식으로 명확하게 데이터를 지칭하지 않는 설명은 이해하기 어렵다.
2. 주석을 적은 후 한번 더 읽고 문장에 문제가 없는지 재확인한다.
3. 함수의 주석이라면 동작을 명확하게 설명해야한다.
4. 해당 함수의 실행 예로 인자값 => 결과를 적는 것 또한 좋은 방법이다.


# 7장 정리

## 읽기 쉽게 흐름제어 만들기

1. if문의 조건에 사용되는 함수는 true를 반환하는 조건하에서 행동하도록 넣는다.
2. 개발자는 항상 로직에서의 정상 시퀀스를 true로 잡되, 예외 사항 또한 true로 생각해야한다.
3. 가독성이 떨어지는 반복문은 되도록 사용하지 않는다.(do_while, goto, 삼항연산자(복잡한 수식일 경우)
4. 함수 중간에 return으로 반환하는 로직으로 예외 처리를 하면 코드가 깔끔해질 수 있다. return을 하나로만 사용하려는 생각은 하지 말자.
