10장. 모르면 답답해지는 네트워크 기초 핵심 요약
1. 왜 네트워크 기초를 알아야 하는가?
   • 문제 해결 능력: 코드 문제가 아니어도 네트워크 설정(IP 오기재, 방화벽 등)으로 인해 API 연결 실패 등이 발생할 수 있음.
   • 실무 사례: 외부 업체와 연동 시, '서버 접속용 IP'와 '서버가 외부로 나갈 때 쓰는 IP'를 혼동하여 연결이 안 되는 경우가 빈번함.
2. 네트워크 기본 용어
   • 노드 (Node): 데이터를 송수신하는 모든 장치 (서버, 노트북, 스마트폰 등).
   • 패킷 (Packet): 데이터 전송 단위. 헤더(발신/수신자 정보)와 페이로드(실제 데이터)로 구성.
   • 라우터 (Router): 서로 다른 네트워크 간에 패킷을 중계/전송하는 장치.
3. IP 주소와 도메인 (DNS)
   • IPv4: 현재 주로 사용하는 32비트 주소 체계 (예: 223.130.192.248).
   • DNS (Domain Name System): 사람이 기억하기 쉬운 도메인 이름(예: naver.com)을 컴퓨터가 이해하는 IP 주소로 변환해주는 시스템.
   ◦ 참고: 하나의 도메인에 여러 IP가 연결될 수 있으며(부하 분산), nslookup 명령어로 확인 가능.
   • Localhost: 자기 자신을 가리키는 주소. IP는 항상 127.0.0.1.
   • Hosts 파일: DNS보다 우선순위가 높은 로컬 도메인 매핑 파일. (테스트 환경 구성 시 유용)
4. IP의 종류 (필수 구분)구분설명특징고정 IP변경되지 않는 IP주로 서버에 사용.
   동적 IP접속 시마다 할당받는 IPDHCP 서버가 할당. 일반 가정/사무실 PC.공인 IP (Public)인터넷 상에서 유일한 주소외부에서 접근 가능.사설 IP (Private)내부 네트워크 전용 주소외부 접근 불가. 내부에서만 유효함.📝 사설 IP 대역 (외워두면 좋음)
   • 192.168.x.x (공유기 등에서 가장 흔함)
   • 10.x.x.x (AWS 등 클라우드 내부망에서 흔함)
   • 172.16.x.x ~ 172.31.x.x
5. 네트워크 연결 및 보안 기술 (NAT & VPN)
   • NAT (Network Address Translation): 사설 IP와 공인 IP를 변환하는 기술.
   ◦ SNAT (Source NAT): 내부(사설 IP)에서 외부로 나갈 때, 공인 IP로 변환. (내부 단말이 인터넷 할 때)
   ◦ DNAT (Destination NAT): 외부(공인 IP)에서 들어올 때, 특정 내부 서버(사설 IP)로 매핑. (포트 포워딩 등)
   • VPN (Virtual Private Network):
   ◦ 외부 네트워크에서 내부 서버 네트워크(사설망)에 안전하게 접속하기 위해 사용.
   ◦ 데이터를 암호화하여 마치 내부망에 있는 것처럼 통신하게 해줌. (재택근무/외부 미팅 시 필수)
6. 전송 프로토콜 (TCP / UDP / QUIC)
   백엔드 개발자는 주로 전송 계층(L4)과 애플리케이션 계층(L7)을 다룹니다.프로토콜특징 장점 단점/이슈용도TCP연결 지향 (3-Way Handshake)신뢰성 높음 (순서 보장, 재전송)상대적으로 느림, HOL 블로킹 문제HTTP, SMTP 등 대부분의 웹 통신UDP비연결형속도 빠름 (오버헤드 적음)데이터 유실 가능성, 순서 보장 XDNS, 스트리밍, 게임QUICUDP 기반 (HTTP/3의 기반)빠르고 신뢰성 있음 (TLS 통합, 멀티플렉싱)TCP의 HOL 블로킹 문제 해결최신 웹 브라우저, CDN 등


   💡실무 Tip
1. 연결 문제 발생 시: 상대방 서버가 나의 어떤 IP(공인 IP vs 사설 IP)를 허용해 줬는지, 나는 어떤 IP로 나가고 있는지(NAT 확인) 체크해 보세요.
2. 프로토콜 확인: 현재 개발 중인 서비스나 자주 쓰는 사이트가 HTTP/3 (QUIC)을 지원하는지 브라우저 개발자 도구의 Network 탭(Protocol 컬럼)에서 확인해 보시면 최신 트렌드를 익히는 데 도움이 됩니다.
