1. NoSQL의 이해 (유형 및 특징)
   NoSQL은 데이터를 저장하는 방식에 따라 크게 4가지로 분류됩니다. 
   각 유형은 고유한 특징과 적합한 용도가 있습니다.

| 유형 | 특징 | 대표 기술 | 주요 용도 |
| :--- | :--- | :--- | :--- |
| **키-값 DB**<br>(Key-Value) | • 가장 단순한 구조 (Map과 유사)<br>• 읽기/쓰기 속도가 빠름<br>• 모든 데이터를 값으로 저장 가능 | Redis,<br>DynamoDB | • 세션 관리<br>• 캐시 (Caching)<br>• 설정(Config) 관리 |
| **문서 DB**<br>(Document) | • JSON 유사 형식 저장<br>• 스키마가 유연함 (속성 추가 용이)<br>• 애플리케이션과 DB의 모델 일치 | MongoDB | • 콘텐츠 관리 (CMS)<br>• 제품 카탈로그<br>• 계층 구조 데이터 |
| **칼럼 패밀리 DB**<br>(Column Family) | • 대량 데이터 저장 및 수평 확장 용이<br>• 여러 칼럼을 그룹으로 묶어 관리<br>• *참고: 분석용 칼럼 기반 DB와는 다름* | Cassandra,<br>HBase | • 채팅 로그 저장<br>• IoT 센서 데이터<br>• 대규모 데이터 처리 |
| **그래프 DB**<br>(Graph) | • 노드(Node)와 엣지(Edge)로 관계 표현<br>• 데이터 간의 '관계' 탐색에 특화 | Neo4j | • 소셜 네트워크<br>• 추천 시스템<br>• 부정 탐지 (FDS) |
💡 CAP 정리와 NoSQL
분산 시스템 이론인 CAP 정리(Consistency, Availability, Partition tolerance)에 따라 NoSQL은 **분할 내성(P)**을 기본으로 하고, **일관성(C)**과 가용성(A) 중 하나를 선택합니다.

CP (일관성 우선): MongoDB (네트워크 분할 시 일관성을 위해 일부 기능 차단)

AP (가용성 우선): Cassandra (일관성이 일시적으로 깨져도 서비스 지속, Eventual Consistency)

⚠️ NoSQL 도입 시 주의사항
트랜잭션: RDBMS 수준의 ACID 트랜잭션이 지원되지 않는 경우가 많으므로 검증이 필요합니다.

데이터 모델: 단순 캐시엔 키-값, 계층형 데이터엔 문서 DB 등 용도에 맞게 선택해야 합니다.

운영 복잡성: 백업, 모니터링 등이 RDBMS보다 복잡할 수 있으며, 팀의 기술 역량을 고려해야 합니다.

Note: "새로운 기술이라고 무작정 도입하지 마라." 필자는 불필요하게 MongoDB를 도입했다가 다시 단순한 구조로 롤백한 경험을 통해 기술 도입의 신중함을 강조합니다.

2. DB로 분산 잠금(Distributed Lock) 구현하기
   여러 서버(노드)에서 동시에 실행되는 스케줄러나 배치 작업이 중복 실행되지 않도록 하기 위해, Redis나 ZooKeeper 대신 RDBMS를 이용하여 분산 잠금을 구현하는 방법입니다.

1) 핵심 원리
   별도의 잠금 테이블(dist_lock)을 생성하여 관리합니다.

SELECT ... FOR UPDATE 구문을 사용하여 동시성 이슈를 제어합니다.

잠금 소유자(owner)와 만료 시간(expiry)을 두어 데드락을 방지합니다.

2) 테이블 구조 (dist_lock)
   name (PK): 잠금의 이름 (락을 구분하는 키)

owner: 잠금을 소유한 주체 (스레드 ID 등)

expiry: 잠금 자동 만료 시간 (장애 발생 시 락 해제용)

3) 잠금 획득 로직 (tryLock)
   트랜잭션을 시작하고 SELECT ... FOR UPDATE로 해당 락의 행(Row)을 조회합니다.

데이터가 없는 경우: INSERT로 행을 생성하며 잠금 획득 (성공)

데이터가 있는 경우:

내 소유인 경우: expiry 시간만 연장 (성공)

남의 소유 + 만료 시간 지남: owner를 나로 변경하고 expiry 갱신 (성공 - 탈취)

남의 소유 + 만료 시간 안 지남: 잠금 획득 실패 (실패)

4) 잠금 해제 로직 (unlock)
   현재 소유자가 자신인지 확인합니다.

만료 시간이 지났는지 확인합니다 (이미 만료되었다면 예외 처리).

owner와 expiry를 NULL로 업데이트하여 잠금을 반납합니다.
