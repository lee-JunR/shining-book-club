# 부록 B NoSQL 이해하기

데이터 요구사항이 다양해지면서 그에 맞춰 데이터 설계 방식도 다양해지고 있다.

- 수천만 회원 간의 연결 관계를 분석해서 친구를 추천한다.
- 대량의 데이터를 실시간으로 수집하고통계를 추출한다.
- 테라바이트에서 페타바이트 이상의 데이터를 처리한다.

RDBMS로도 이런 요구를 처리할 수는 있지만 더 알맞은 기술을 찾아야 할 때도 있다.

과거에는 No-SQL은 관계형 데이터베이스의 기술을 배제한 데이터베이스였지만 시간이 지남에 따라 NoSQL 기술도 SQL 과 유사한 언어를 지원하기 시작했다.

NoSQL 을 사용하는 이유

- 대용량 데이터
  - 수평 확장을 통해 Cassendra나 HBASE의 경우 데이터 저장 용ㅇ량을 늘릴 수 있다.
- 고속 읽기 쓰기 성능
  - 분산처리나 높은 성능과 같은 특징을 제공하기 위해 ACID 요건 중 일부를 지원하지 않는 NoSQL 도 있다.
- 특정한 요구사항에 맞는 설계
- 비정형 데이터 처리 또는 유연한 스키마

수평 확장을 통한 대용량 데이터 처리에 강점이 있는 NoSQL 도 있다.

### NoSQL 종류

NoSQL 을 나누는 다양 한 기준

- 키-값 DB
- 문서 DB
- 칼럼 패밀리 DB
- 그래프 DB
