### 프로세스 확인하기

서버의 정상 동작을 확인하기 위해 프로세스가 정상적으로 동작하는지 확인하려면 프로세스가 사용하는 CPU나 메모리를 확인해야한다. 이를 위해서 사용하는 명령어는 다음과 같아.

문제를 일으키는 프로세스는 보통 CPU나 메모리 사용량을 많이잡아먹는 경우가 많으므로 다음과 같은 명령어를 알아두면 좋을 것이다.

**`htop`**: 시스템의 프로세스와 리소스 사용량을 실시간으로 모니터링하는 대화형 도구입니다.

**`ps aux --sort -rss | head -n 6`**: 메모리 사용량(RSS)을 기준으로 내림차순 정렬하여 상위 6개 프로세스를 보여줍니다.

### 프로세스 종료하기

문제를 일으키는 프로세스를 확인했다면 이제 종료해야한다.

프로세스를 종료할 때에는 `kill` 명령어를 사용한다.

문법은 다음과 같다

```protobuf
kill 옵션 PID
```

옵션은 보통

- -15, -s SIGTERM, -TERM : 안전한 종료
- -9, -s SIGKILL 또는 -KILL : 강제 종료

를 사용한다.

## 백그라운드 프로세스

명령어 뒤에 &를 붙이면 프로세스가 백그라운드로 실행되므로 다른 포그라운드 작업을 동시에 할 수 있다.

하지만 이때에는 사용자가 로그아웃하면 터미널에서 실행한 백그라운드 프로세스가 함께 종료될 수 있다.

터미널을 종료해도 백그라운들 프로세스가 실행되게 하려면 `nohup` 명령어를 사용하자.

## 디스크 용량 관리

디스크 사용률을 확인하려면 `df -h` 명령어를 사용한다. 어떤 파티션의 사용률이 높은지 확인할 수 있다.

삭제 대상을 찾을 때는 어떤 디렉토리가 많은 용량을 차지하는지 확인해야 한다. 이때는 `du -sh` 명령어를 사용한다.

`-s` 옵션은 하위 디렉토리의 용량 합과 파일 크기를 보여준다. `-h` 옵션은 바이트 대신 사람이 읽기 쉬운 단위로 표시해준다.

파일을 제외하고 하위 디렉토리의 용량 합만 구하고 싶다면 `du -sh */` 명령어를 사용한다.

오래된 파일을 삭제할 때에는 find 명령어를 사용한다.

`find ./logs -mtime +29 -type f -delete`

해당 명령어를 사용하면 logs 디렉토리에서 30일 이전 파일을 삭제할 수 있다.

## 파일 디스크럽터 제한

프로세스는 데이터 입출력이 필요할 때 OS로부터 파일 디스크립터를 할당받는다.

OS는 시스템 수준에서 생성할 수 있는 파일 디스크립터 수를 제한한다.

프로세스는 제한된 개수를 초과해서 사용할 수 없다.

따라서 파일과 소켓을 합쳐 파일 디스크립터의 수가 제한을 초과하지 않도록 확인하고 늘려야 한다.

`ulimit -a` 명령어로 사용자의 파일 디스크립터 개수 제한을 확인할 수 있다.

사용자가 파일 디스크립터 제한을 변경하려면 `ulimit -n 개수` 명령어를 사용하면 된다.

하지만 이 명령어는 현재 세션에만 적용되므로 다른 세션에는 적용되지 않으며, 로그아웃 후 다시 접속하면 기본값으로 돌아간다.

기본 파일 디스크립터 제한을 영구적으로 변경하려면 `/etc/security/limits.conf` 파일을 수정한다.

`* soft nofile 10000`

`* hard nofile 10000`

여기서 `soft`는 사용자가 변경할 수 있는 제한이고, `hard`는 시스템이 허용하는 최대 제한이다.

`*`는 모든 사용자에게 적용한다는 뜻이다.

`systemd`로 실행되는 서비스의 파일 디스크립터 제한은 `systemctl show` 명령어로 확인할 수 있다.

해당 서비스는 `/etc/systemd/system.conf` 파일의 `DefaultLimitNOFILE` 값을 기본 파일 디스크립터 제한으로 사용한다.

`systemctl daemon-reload` 명령으로 설정을 반영한다.

각 서비스마다 다른 값을 지정할 수 도 있다.

개별 서비스 파일 설정에 limitNOFILE 설정을 추가하면 된다. (해당 설정은 /etc/systemd/system 에 존재한다.)

한 프로세스가 가질 수 있는 파일 디스크럽터 수 제한도 확인해야한다. 이건

`sysctl fs.nr_open` 명령어로 확인할 수 있다.
