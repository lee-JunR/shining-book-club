- 기본 구성 요소의 이름
  - 노드 : 데이터 주고받는 장치 (서버, pc)
  - 패킷 : 네트워크를 통해 전송되는 데이터의 단위
  - 라우터: 서로 다른 네트워크에 속한 노드는 직접 통신 x → 이때 패킷 경로를 찾아 전송해주는 장치

## 네트워크 연결 기술 (NAT & VPN)

IPv4 주소 부족 문제를 해결하고 보안을 유지하기 위해 NAT 기술을 사용한다.

**NAT (Network Address Translation)**
사설 IP와 공인 IP를 상호 변환하는 기술이다

- **SNAT**: 내부(사설 IP)에서 외부로 나갈 때 공인 IP로 변환한다. (소스 IP 변경)
- **DNAT**: 외부(공인 IP)에서 내부 서버로 들어올 때 사설 IP로 변환한다. (목적지 IP 변경)

**VPN (Virtual Private Network)**
외부(카페, 집)에서 사설 IP로 구성된 회사나 서버 네트워크에 안전하게 접속해야 할 때가 있다. 이때 사용하는 것이 암호화된 가상 사설망인 VPN이다.

### 전송 프로토콜 (TCP, UDP, QUIC)

데이터를 어떻게 전송할지에 대한 약속인 프로토콜도 상황에 맞춰 선택해야 한다.

**TCP (신뢰성 중시)**
연결 기반(3-Way Handshake)으로 데이터의 순서와 전송을 보장한다. 하지만 확인 과정 때문에 속도가 상대적으로 느리다. HTTP, SMTP 등에서 사용한다.

**UDP (속도 중시)**
연결 과정 없이 데이터를 던지는 방식이다. 빠르지만 데이터 유실이나 순서 뒤바뀜이 발생할 수 있다. 스트리밍, 게임, DNS 등에서 사용한다.

**QUIC (신뢰성+속도)**
UDP를 기반으로 하지만 TCP의 신뢰성 기능을 포함하고 TLS 보안을 통합했다. 전송 속도를 비약적으로 높인 차세대 프로토콜
