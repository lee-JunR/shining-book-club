## 성능 테스트 종류

- **부하(Load) 테스트**  
  예상되는 정상 트래픽 상황에서 시스템이 안정적으로 동작하는지 확인한다.

- **스트레스(Stress) 테스트**  
  점진적으로 부하를 증가시켜 시스템의 최대 성능과 한계를 확인한다.

- **지속 부하(Soak) 테스트**  
  장시간 일정한 부하를 가해 메모리 누수나 성능 저하 여부를 검증한다.

- **스파이크(Spike) 테스트**  
  트래픽이 급격히 증가하거나 감소할 때 시스템의 반응성과 안정성을 확인한다.

---

## 포화점과 버클존

- **포화점(Saturation Point)**  
  처리량이 더 이상 증가하지 않으며, 응답 시간이 급격히 저하되기 직전의 최대 처리량 지점이다.

- **버클존(Buckle Zone)**  
  포화점을 지나 시스템 성능이 눈에 띄게 꺾이기 시작하는 구간이다.

> 버클(Buckle)은 구조물이 압축 하중을 견디지 못하고 구부러지는 현상에서 유래한 용어이다.

## 주요 측정 지표

### 응답 시간(Response Time)
- 평균, 최소, 최대
- 중앙값
- 95%, 99% 백분위

### 처리량(Throughput)
- TPS(초당 트랜잭션 수)
- 최대 / 평균 / 최소 값 측정

### 에러율(Error Rate)
- 전체 요청 중 실패 비율

### CPU 사용률
- 시스템 병목 지점을 판단하는 주요 지표

## 성능 테스트 실행 시 주의 사항
- 부하 발생기와 테스트 대상 시스템을 분리한다.
- 서버 설정 제한 상태에서 테스트하지 않도록 주의한다.
- 부하 규모에 맞게 DB 커넥션 풀을 미리 설정한다.
- 운영 환경과 동일한 네트워크 구성 시 주의가 필요하다.
- 외부 서비스 연동은 실제 서비스를 호출하지 않도록 한다.
- 테스트 환경은 운영 환경과 최대한 동일하게 구성한다.

## NoSQL 이란
- 특정 요구사항에 맞춰 데이터를 저장하고 조회하기 위한 데이터베이스
- 대용량 데이터와 분산 처리에 적합
- 고속 읽기·쓰기 성능 제공
- 비정형 데이터 및 유연한 스키마 지원
- 수평 확장 구조 사용

### RDBMS vs NoSQL
- RDBMS: 고정된 스키마, 관계형 모델
- NoSQL: 유연한 스키마, 용도 중심 데이터 모델
