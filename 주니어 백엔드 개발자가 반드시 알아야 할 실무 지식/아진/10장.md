## 노드, 네트워크, 라우터
- 노드: 데이터를 송수신하는 모든 장치  
  (휴대폰, 노트북, 서버 장비 등)
- 네트워크: 각 노드가 서로 데이터를 주고받기 위해 연결된 시스템
- 패킷: 노드가 네트워크를 통해 전송하는 데이터의 단위
    - 헤더: 발신자·수신자 정보 포함
    - 페이로드: 실제 데이터
    - 데이터는 여러 패킷으로 나뉘어 전송됨
- 라우터: 네트워크 간에 패킷을 전달하는 역할

---

## IP 주소와 도메인
- IP 주소: 네트워크에서 각 노드를 구분하기 위해 사용하는 주소

### IPv4와 IPv6
- IPv4: 32비트 주소 체계 (약 43억 개)
- IPv6: 128비트 주소 체계
- 현재도 IPv4가 주로 사용됨
- IP 주소 고갈 문제는 **사설 IP와 NAT**로 완화됨

### DNS와 도메인 구조
- DNS: 도메인 이름과 IP 주소를 변환하는 체계
- 도메인은 계층 구조를 가짐
    - 점(.)으로 구분
    - 오른쪽이 상위, 왼쪽이 하위
- 최상위 도메인
    - 일반: com, org, net, gov, biz, tech 등
    - 국가: kr, jp, au 등
- 일반 최상위 도메인
    - 2차 도메인이 주요 이름 (예: naver.com)
    - 3차 이후는 용도별 구분 (cafe.naver.com)
- 국가 최상위 도메인
    - 2차 도메인이 미리 정의됨 (co.kr, ac.kr)

## IP 주소의 종류

### 고정 IP와 동적 IP
- 고정 IP
    - IP 주소가 변하지 않음
    - 주로 서버에서 사용
- 동적 IP
    - 네트워크 연결 시 IP 할당
    - DHCP 서버가 IP, 게이트웨이, 서브넷 마스크, DNS 정보 제공
    - 가정용 공유기 환경에서 주로 사용

### 공인 IP와 사설 IP
- 공인 IP
    - 인터넷에서 접근 가능한 주소
    - 방화벽으로 막지 않으면 외부 접근 가능
- 사설 IP
    - 내부 네트워크 전용 주소
    - 외부에서 직접 접근 불가

---

## NAT
- NAT(Network Address Translation): 네트워크 주소 변환 기술
- 사설 IP ↔ 공인 IP 변환 담당
- 주로 인터넷에 연결된 라우터가 수행
- SNAT
    - 출발지 IP 주소 변환
- DNAT
    - 목적지 IP 주소 변환

---

## VPN
- VPN(Virtual Private Network)
- 공용 네트워크를 통해 사설 네트워크에 안전하게 접근하는 기술
- 서로 다른 네트워크 간 암호화된 연결 제공

---

## 프로토콜

### 프로토콜과 계층
- 프로토콜: 네트워크에서 데이터를 주고받기 위한 규칙
- TCP/IP 모델
    - 애플리케이션
    - 전송
    - 인터넷
    - 네트워크 인터페이스

### 전송 계층 프로토콜
#### TCP
- 연결 기반 프로토콜
- 패킷 순서 보장
- 유실 시 재전송
- 신뢰성 높음, 상대적으로 느림
- 사용 예: HTTP, SMTP

#### UDP
- 비연결 기반 프로토콜
- 순서 보장 없음
- 빠르지만 신뢰성 낮음
- 사용 예: DNS, VoIP, 게임

#### QUIC
- UDP 기반
- TCP의 연결 관리 기능 제공
- TLS 통합
- 멀티플렉싱 지원
- HTTP/3에서 사용
