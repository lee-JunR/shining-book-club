## NoSQL 이란
### 1. 키-값 DB
- 예: DynamoDB, Redis
- 주요 용도
    - 세션 관리
    - 캐시
    - 설정 관리

### 2. 문서 DB
- 예: MongoDB
- JSON과 유사한 문서 형태로 데이터 저장
- 스키마가 고정되어 있지 않음
- 주요 용도
  - 콘텐츠 관리
  - 제품 카탈로그 (다양한 메타데이터)

#### 3. 컬럼 패밀리 DB
- 예: **Cassandra, HBase**
- 대규모 분산 데이터 처리에 적합
- 주요 용도
  - 채팅 메시지
  - IoT 데이터
- 참고
  - **컬럼 기반 DB(ClickHouse 등)** 는 분석(OLAP) 목적

#### 4. 그래프 DB
- 예: **Neo4j**
- 노드와 엣지로 관계를 표현
- 주요 용도
  - 소셜 네트워크
  - 추천 시스템
  - 부정(이상) 탐지

---

### NoSQL 도입 시 고려사항
1. 트랜잭션 지원 여부
2. 데이터 모델이 요구사항에 맞는지
3. 확장성과 성능
4. 운영 및 개발 역량 확보

---

### CAP 이론 정리
- 분산 시스템은 아래 3가지를 **동시에 모두 만족할 수 없다**
  - **일관성(C)**: 모든 노드가 같은 데이터를 봄
  - **가용성(A)**: 모든 요청에 응답
  - **분할내성(P)**: 네트워크 분리 상황에서도 동작

- 분산 시스템(NoSQL)은 기본적으로 **P를 전제로** 함
  - AP: 가용성 우선
  - CP: 일관성 우선

#부록 C. DB로 분산 잠금 구현하기 (핵심 흐름 정리)

### 분산 잠금 개념
- 여러 서버/스레드가 **하나의 자원에 동시에 접근하지 못하도록 제어**하는 메커니즘
- DB의 트랜잭션과 행 잠금을 이용해 구현 가능

### 분산 잠금 동작 흐름
1. 트랜잭션 시작
2. 선점 잠금 쿼리로 행 점유 시도
3. 행이 없으면 새 잠금 데이터 생성
4. owner가 다르고 expiry가 남아 있으면 실패
5. expiry가 지났으면 owner 갱신 후 획득
6. owner가 같으면 expiry 갱신
7. 트랜잭션 커밋 → 잠금 성공
8. 커밋 실패 시 잠금도 실패
