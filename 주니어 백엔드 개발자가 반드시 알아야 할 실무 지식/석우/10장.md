# 10장 정리 -

## 10장 모르면 답답해지는 네트워크 기초.

- 네트워크와 라우터
- IP 주소와 도메인
- NAT
- VPN
- 프로토콜

## ** 최소한 알고 있어야 할 서버 지식 지식 정리 **

### 네트워크와 라우터

데이터를 송수신하는 모든 장치를 노드(Node)라고 하며 여기에는 휴대폰, 노트북, 서버 장비 등이 포함된다.

이러한 노드들이 서로 데이터를 주고받기 위해 연결된 시스템을 네트워크(Network)라고 한다.

노드가 네트워크를 통해 전송하는 데이터의 단위를 패킷(Packet)이라고 하며 패킷은 발신자와 수신자 정보가 담긴 헤더와 실제 데이터가 담긴 페이로드로 구성된다.

서로 다른 네트워크에 속한 노드는 직접 통신할 수 없으며, 이때 라우터(Router)가 네트워크 간에 패킷을 전송하는 역할을 수행한다.

### IP 주소와 도메인

네트워크에서 각 노드를 구분하기 위해 사용하는 주소를 IP 주소라고 한고 일반적으로 1바이트(8비트) 4개의 숫자 블록으로 구성된 IPv4 주소(예: 223.130.192.248)를 사용한다.

- 도메인 이름(Domain Name): IP 주소를 기억하기 쉽게 붙인 이름이며, DNS(Domain Name System) 서버를 통해 도메인 이름이 IP 주소로 변환

- 루프백 주소 (localhost): 컴퓨터에서 자기 자신을 참조할 때 사용하는 특별한 주소로, IP 주소는 127.0.0.1 이다.

- 고정 IP / 동적 IP: 서버 IP는 보통 바뀌지 않는 고정 IP를 사용하지만, 동적 IP는 DHCP 서버를 통해 할당

- 공인 IP / 사설 IP: 공인 IP 주소는 인터넷에서 접근 가능하며, 사설 IP 주소는 네트워크 내부에만 적용되어 외부에서 접근할 수 없습니다. 사설 IP 주소의 사용 덕분에 IPv4 주소 고갈이 방지

### NAT (Network Address Translation)

NAT는 네트워크 주소를 변환하는 기술입니다. 이는 사설 IP와 공인 IP 주소 간의 변환을 담당하며, 주로 라우터와 같은 네트워크 장비가 수행

- SNAT (Source NAT): 내부 네트워크에서 나가는 패킷의 사설 IP를 공인 IP로 변환

- DNAT (Destination NAT): 공인 IP로 들어온 패킷의 목적지 IP를 사설 IP로 변환합니다. 서버 노드가 보안 등의 이유로 사설 IP를 사용할 때, 외부에서 공인 IP로 접근하면 DNAT를 통해 서버 노드에 패킷이 전송

### VPN (Virtual Private Network)

VPN은 가상 사설 네트워크(Virtual Private Network)의 약자

이는 인터넷과 같은 공용 네트워크에서 서로 다른 네트워크 간에 암호화된 연결을 제공한다.

VPN을 사용하면 두 네트워크가 마치 하나의 사설 네트워크에 존재하는 것처럼 연결될 수 있으며 개발자는 이를 통해 서버 노드나 DB와 같이 사설 IP를 사용하는 서버 네트워크에 안전하게 접근할 수 있다.

### 프로토콜

프로토콜은 네트워크 상에서 두 노드가 데이터를 주고받기 위해 정의한 규칙

- TCP (Transmission Control Protocol): 연결 기반 프로토콜이며, 3-Way Handshake를 통해 연결을 수립합니다. 신뢰성이 장점이며 패킷 순서를 보장하고 유실 시 재전송합니다. HTTP, SMTP 등 많은 프로토콜이 TCP를 기반으로 동작한다.

- UDP (User Datagram Protocol): 연결 과정 없이 바로 데이터를 전송합니다.
  속도는 빠르지만 데이터가 정상적으로 전송되었는지 알 수 없고 순서도 보장하지 않는다.

- QUIC: UDP를 기반으로 하며 TCP의 연결 관리 기능과 TLS(암호화)를 통합하여 신뢰성을 유지하면서도 빠른 통신을 제공합니다. HTTP/3 프로토콜은 QUIC을 기반으로 사용한다.

#### 결론

백엔드 개발자가 네트워크 관련 지식이 전혀 없다면 네트워크 관련 문제가 발생했을 때 쉽게 해결할 수 있는 문제도 오랜 시간 동안 처리하지 못하게 될 수 있다.

예를 들어 서버에서 외부 API를 호출할 때 서버가 2개의 공인 IP를 사용하고 있음에도 불구하고 외부 통신용 IP가 아닌 외부 연결용 IP를 공유하여 연결에 실패했던 사례처럼 기초 지식의 부족은 사소한 문제로 인한 긴 처리 지연을 유발할 수 있다.

따라서 주니어 백엔드 개발자는 코드를 구동하는 환경을 이해하고 외부 연동 문제를 신속하게 해결하기 위해 IP 주소, 도메인, NAT, VPN의 개념 및 TCP/UDP와 같은 핵심 프로토콜의 원리를 숙지해야 한다고 생각한다.
